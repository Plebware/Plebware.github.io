# 🔑 Google Account and Google Drive Management Blueprint

## Purpose

This document provides a structured plan for managing multiple Google Accounts and their associated Google Drives.

The objectives are:

- 🔑 Prevent duplication of files
- 🔑 Keep projects separated by purpose
- 🔑 Simplify backups
- 🔑 Improve long-term organisation
- 🔑 Support PlebMachine and PlebWare development
- 🔑 Create a clear publishing workflow
- 🔑 Protect important documents

---

# 🔑 Master Rule

Every Google Account should have a clearly defined role.

Avoid storing the same file in multiple Google Drives unless it is a deliberate backup.

Think of each Google Drive as a separate department in an organization.

# 🔑 GitHub Relationship

## PlebWare GitHub

Store:

- Articles
- Documentation
- Tutorials
- Devotions
- Public graphics

### Do Not Store

- Personal documents
- Medical records
- Financial records
- Password information
- Legal documents

GitHub should be treated as a public library.

Google Drive should be treated as your filing cabinet.

## Daily

- Devotion archives

- Complete Google Drive exports
- External SSD backup

- Full archive snapshot
- Long-term preservation archive
